## 核心领域模型要点（系统认知 2.5，下次恢复用）
- 层级链：项目 → 拆主材(按主材履约) → 每主材按**模板**生成多个主材任务 `task_dispatch`(task_type 区分) → 每任务内多个有序节点 `task_dispatch_node`(展示流转进度)。
- `task_dispatch`(主材任务实例表)：`process_status` 4态(1未激活/2激活未完成/3激活已完成/4暂停激活)；字段 supplier_code/order_no/plan_activate_time/node_task(有序序列)/current_node_type。
- `task_dispatch_node`(任务节点实例表)：`node_type` 阶梯 0默认/20通知可启动/40通知启动/60启动/65自检验收/80启动完成/85业主确认；`process_status` 3态(无暂停)；记录激活时间(plan_activate_time/estimated_time/start_time/current_node_type)与处理人(executor_type/executor_id/submit_by)。
- 激活区别：任务级(一次SKU履约,4态,触发 activateIfNeeded/afterCreate,风险两次激活竞争) vs 节点级(一道工序,3态,由任务推进驱动 current_node_type 前移,风险前置拦截/restart)。
- 模板驱动：依据 品类×城市×套餐×配置模式 匹配模板，模板 processChain 决定节点序列（标准/含变更/整装 三例；维度表述为代码推断，非用户逐字确认）。
