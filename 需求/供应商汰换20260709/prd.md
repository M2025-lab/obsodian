# **【背景】**

- 项目生成后，系统会基于“品类-供应商”维度生成货前服务任务，例如复尺、报价变更、测量等；货前服务完成后，会触发后续下货单流程
- 在货前服务及货单流转过程中，可能出现供应商汰换场景。供应商汰换由上游系统发起，并明确告知汰换方式，包括：
    - 常规汰换
    - 紧急汰换 + 立即切换
    - 紧急汰换 + 不立即切换

- 任务侧需要根据上游传入的汰换方式、原供应商、目标新供应商、品类信息，以及当前“品类-供应商”维度下货前任务状态，进行任务处理
- 本 PRD 聚焦：**供应商汰换后，货前服务任务如何识别、取消、生成或保留**

# **【目标】**

本次仅处理供应商汰换后，货前任务层的识别和处理，包括：

- 监听上游供应商汰换消息及汰换方式
- 识别对应品类-老供应商下的货前任务状态
- 根据场景取消老供应商未完成货前任务
- 根据场景生成新供应商货前任务
- 记录处理结果及日志

# **【范围】**

- 2.5所有业务模式（整装、翻新全案、新团装、纯零售）
- 全国（被窝+圣都）

  

  

# **【需求详述】**

## **整体处理逻辑：系统监听到供应商汰换消息后，按以下步骤处理：**

- 解析汰换消息（上游明确需汰换场景才触发，否则任务侧无感知），及对应履约结果，获取“项目订单、品类、老供应商、新供应商、汰换方式（汰换方式如下3种）”
    - 普通汰换：新商或老商履约
    - 紧急汰换+立即切换：新商或老商履约
    - 紧急汰换+不立即切换：新商或老商履约
- 按“项目-品类-老供应商”查询当前货前任务
    - 判断老供应商货前任务是否存在及是否完成
- 根据汰换方式和货前任务状态，决定：
    - 老供应商货前任务是否处理
    - 新供应商货前任务是否生成

## **各场景下的货前任务处理规则**

- 任务角色：指货前服务的作业角色，非通知角色
- 按新商配置及用量判断是否生成
    - 新商有配置及有用量则生成；新商无配置或无用量则不生成
    - 是否去重、是否可重复生成的逻辑保持不变
- 老商任务-未生成：当前无有效老商任务，包含已取消、未生成（无需细拆分未生成原因）
- 以下都是「单一货前任务」判断，若该订单该品类存在多个货前任务，则需要分别判断处理
- 角色场景
    
    - 角色场景明细对应
        
        |a：新老商任务角色一致，且均为非供应商|老商角色=非供应商、新商角色=非供应商|
        |b：非上述角色场景，包含3种情况|老商角色=供应商、新商角色=非供应商<br><br>老商角色=非供应商、新商角色=供应商<br><br>老商角色=供应商、新商角色=供应商|
        
    - 老商/新商角色判断
        - 老商角色判断：有老商任务，看任务角色；无老商任务，看老商配置；都没有，则归属**分类b**
        - 新商角色判断：新商统一看新商配置；无新商配置，则归属**分类b**
        - 总结：
            - 老商新商任务角色**都可判断时**，再具体区分分类a或b
            - 老商新商任务角色**至少一个不可判断时**，则默认归属**分类b**

## **第一层判断（上游输出结果****）**

![[需求/供应商汰换20260709/fig/Pasted image 20260722104906.png]]

- 解析汰换消息（上游明确需汰换场景才触发，否则任务侧无感知），及对应履约结果，获取“项目订单、品类、老供应商、新供应商、汰换方式（汰换方式如下3种）”
    - 普通汰换：新商或老商履约
    - 紧急汰换+立即切换：新商或老商履约
    - 紧急汰换+不立即切换：新商或老商履约

  

## **第二层判断：若结果为”老商履约“-则无需区分角色场景及任务状态，所有老商任务都不做处理、新商任务都无需新生成**

- 整体逻辑：若为“老商履约”，则老商任务都不做处理，新商任务都不新生成

|   |   |   |   |
|---|---|---|---|
|角色场景|老商任务状态|老商任务处理|新商任务处理|
|不区分ab|已生成未完成|不做处理|不新生成|
|已完成|不做处理|不新生成|
|已取消|不做处理|不新生成|
|未生成|不做处理|不新生成|

## **第二层判断：若结果为”新商履约“-则需区分角色场景及任务状态，进行任务处理（无需再判断货单）**

- 整体逻辑：若为“新商履约”
    - 角色场景a：默认认可老商任务结果，不处理老商任务
        - 老商任务状态有效，则不再生成新商任务
        - 老商任务状态无效/无任务，则按实际情况生成新商任务
    - 角色场景b：默认不认可老商任务结果
        - 老商任务在途，则取消老商任务，按实际情况生成新商任务
        - 老商任务非在途，则不处理老商任务，按实际情况生成新商任务

### 1、「常规汰换」（新商履约）

- 场景说明：
    - 常规汰换下，系统主要根据老商货前任务是否完成，判断是否需要由新商重新承接；整体处理相对保守：已完成任务默认保留，不重复生成新商任务；未完成任务按角色场景判断是否取消并重建
    - 规则处理枚举：
        
        |   |   |   |   |
        |---|---|---|---|
        |角色场景|老商任务状态|老商任务处理|新商任务处理|
        |a：新老商任务角色一致，且均为非供应商<br><br>老商角色=非供应商、新商角色=非供应商|已生成未完成|不做处理|不新生成|
        |已完成|不做处理|不新生成|
        |已取消|不做处理|按新商配置及用量判断是否生成|
        |未生成|不做处理|按新商配置及用量判断是否生成|
        |b：非上述角色场景，包含以下3种情况<br><br>老商角色=供应商、新商角色=非供应商<br><br>老商角色=非供应商、新商角色=供应商<br><br>老商角色=供应商、新商角色=供应商|已生成未完成|取消老商任务|按新商配置及用量判断是否生成|
        |已完成|不做处理|按新商配置及用量判断是否生成|
        |已取消|不做处理|按新商配置及用量判断是否生成|
        |未生成|不做处理|按新商配置及用量判断是否生成|
        

### 2、「紧急汰换 + 立即切换」（新商履约）

- 场景说明：
    - 该场景下，上游已明确后续履约立即切换至新商，货前任务层需避免未完成任务继续由老商执行；除非新老商任务角色一致且均为非供应商，否则未完成老商任务需取消，并结合货单状态判断新商是否重新生成任务
    - 规则处理枚举：
        
        |角色场景|老商任务状态|老商任务处理|新商任务处理|
        |---|---|---|---|
        |a：新老商任务角色一致，且均为非供应商<br><br>老商角色=非供应商、新商角色=非供应商|已生成未完成|不做处理|不新生成|
        |已完成|不做处理|不新生成|
        |已取消|不做处理|按新商配置及用量判断是否生成|
        |未生成|不做处理|按新商配置及用量判断是否生成|
        |b：非上述角色场景，包含以下3种情况<br><br>老商角色=供应商、新商角色=非供应商<br><br>老商角色=非供应商、新商角色=供应商<br><br>老商角色=供应商、新商角色=供应商|已生成未完成|取消老商任务|按新商配置及用量判断是否生成|
        |已完成|不做处理|按新商配置及用量判断是否生成|
        |已取消|不做处理|按新商配置及用量判断是否生成|
        |未生成|不做处理|按新商配置及用量判断是否生成|
        

### 3、「紧急汰换 + 不立即切换」（新商履约）

- 场景说明：
    - 该场景下，上游已明确部分链路暂不立即切换，货前任务层不主动扩大切换范围；未完成任务按规则处理；已完成任务需结合货单状态判断是否仍由老商履约，避免重复生成新商任务
    - 规则处理枚举：
        
        |角色场景|老商任务状态|老商任务处理|新商任务处理|
        |---|---|---|---|
        |a：新老商任务角色一致，且均为非供应商<br><br>老商角色=非供应商、新商角色=非供应商|已生成未完成|不做处理|不新生成|
        |已完成|不做处理|不新生成|
        |已取消|不做处理|按新商配置及用量判断是否生成|
        |未生成|不做处理|按新商配置及用量判断是否生成|
        |b：非上述角色场景，包含以下3种情况<br><br>老商角色=供应商、新商角色=非供应商<br><br>老商角色=非供应商、新商角色=供应商<br><br>老商角色=供应商、新商角色=供应商|已生成未完成|取消老商任务|按新商配置及用量判断是否生成|
        |已完成|不做处理|按新商配置及用量判断是否生成|
        |已取消|不做处理|按新商配置及用量判断是否生成|
        |未生成|不做处理|按新商配置及用量判断是否生成|
        

### 关于新生成任务的激活

- 若可计算激活时间
    - 当前时间<激活时间：不处理激活，等待激活时机
    - 当前时间>=激活时间：直接处理为激活
- 若不可计算激活时间
    - 直接处理为激活
