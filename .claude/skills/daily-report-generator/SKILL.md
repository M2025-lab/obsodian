# 日报生成器

从个人日报记录文件中提取当日工作内容，按照指定格式生成结构化日报表格。

## 使用场景

当用户要求"梳理今天的工作"、"生成日报"、"整理日报"、"输出日报"，或需要从日报记录中总结当日工作时使用。

## 工作流程

### 1. 定位日报记录文件

默认读取路径：`/Users/mirror/wiki/日报与周报/日报记录-个人.md`

如果用户指定了其他文件，按用户指定路径读取。

### 2. 提取当日工作内容

从日报记录中全面提取当日所有工作内容，包括但不限于：
- 显式列出的"今日计划"条目
- 长文本段落中的工作细节
- 输出的技术文档、分析总结
- Bug修复、代码改动
- 业务逻辑梳理
- 问题排查过程

**不要只提取用户整理的条目，要从所有长文本中总结归纳。**

### 3. 生成日报三部分

#### 今日计划
- 每条用**一段话**描述（不是短句，是完整的句子）
- 概括该项工作的核心内容和产出
- 条目数量根据实际工作内容调整，通常3-5条

#### 明日计划
- 基于当日工作进展推导后续工作
- 每条简洁明了
- 条目数量3-4条为宜

#### Issues
- 格式为编号列表：`1. **体会**：内容`、`2. **思考**：内容`
- 从当日工作细节中提炼真实的体会和思考
- 体会：工作中的感悟、发现、经验总结
- 思考：对问题的分析、优化方向、改进想法
- **控制在5条以内**
- 可以加入具体工作细节，避免空泛

### 4. 输出格式

使用 Markdown 表格输出，三行三列：

| 分类 | 内容 |
|:--|:--|
| **今日计划** | 1. xxx<br>2. xxx<br>3. xxx |
| **明日计划** | 1. xxx<br>2. xxx<br>3. xxx |
| **Issues** | 1. **体会**：xxx<br>2. **体会**：xxx<br>3. **思考**：xxx<br>4. **思考**：xxx |

## 提取要点

### 今日计划提炼原则
- 合并同类项，相关工作整合为一条
- 每条一段话，说清楚"做了什么 + 结果/产出"
- 从技术细节中抽象出业务价值
- 涵盖代码开发、逻辑梳理、问题修复、文档输出等

### Issues提炼原则
- 体会：从具体工作中得到的经验、认知、发现
- 思考：对现有方案的反思、优化方向、未来改进
- 每条要有具体细节支撑，不写空话套话
- 优先选有深度、有价值的感悟
- 体会和思考可以穿插，不强制分区
