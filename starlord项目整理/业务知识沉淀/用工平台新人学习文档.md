用工平台主要是用来在工地协调人和材料的，负责协调什么**人**在什么样的**工地**用什么**工具**对什么**材料**按照**工序**施加**工艺**

用工平台的系统架构图，展示了用工平台（Labor Platform）的模块划分及其与上游系统 Cube、下游子系统的关系。用工平台分为"管工地"（工程管理）和"管工人"（服务者管理）两大域。工地侧管打卡、施工包、复尺、播报、质检；工人侧管档案、派单、评价、结算。质检结果红线穿透到工人绩效，形成质量闭环。

## 结构图说明
1. **主链路**
外部统一入口 `cube` 流入核心模块「用工平台」，用工平台分为两大业务分支：**工地工程管理**、**服务者管理**。
2. **工地工程管理模块**
- 附属虚线关联「供应商」；
- 五大子功能：打卡、施工包、复尺数据、播报、质检；
- 打卡通过中间层 `activity` 分流为品控/项目经理打卡、工人打卡；
- 播报通过中间层 `artisan-broadcast` 分流为项目经理播报、摄像头、工人播报。
3. **服务者管理模块**
包含4个子模块：服务者中心、派单、服务者评价（绩效管理）、服务者结算。
4. **关键回流逻辑**
质检结果红色回流至「服务者评价（绩效管理）」，实现施工质量与工人绩效联动考核。
5. **样式还原**
- 供应商使用虚线框；
- 质检到评价的连线为红色粗线；
- 所有节点文字、层级、分支关系1:1匹配原图。
### 工地工程管理

负责项目现场的施工过程管理，包含5个子模块：

| 模块     | 功能                 | 下游系统/场景                                        |
| -------- | -------------------- | ---------------------------------------------------- |
| 打卡     | 人员到场/离场打卡    | → activity 系统 → 品控/项目经理打卡、工人打卡        |
| 施工包   | 施工包全生命周期管理 | （我们之前详细讨论过）                               |
| 复尺数据 | 现场复尺、工程量确认 | —                                                    |
| 播报     | 工地进度播报         | → artisan-broadcast → 项目经理播报、摄像头、工人播报 |
| 质检     | 质量检查、验收       | → 红线连接至 服务者评价（质检结果影响工人绩效）      |

### 服务者管理

负责工人/服务者的档案、调度、评价、结算，包含4个子模块：

| 模块                   | 功能                       | 说明                   |
| ---------------------- | -------------------------- | ---------------------- |
| 服务者中心             | 工人档案、入驻、认证、培训 | 服务者管理的基础数据   |
| 派单                   | 工人派单、约工、产能分配   | 施工包生成后的核心动作 |
| 服务者评价（绩效管理） | 工人评分、绩效计算         | ← 接收质检结果（红线） |
| 服务者结算             | 工人工资、阶段结算         | 与业财系统对接         |
